# mk-official-notes

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文件形式保存，不依赖特定后端服务，可直接在浏览器中打开或通过 GitHub Pages 等静态托管服务访问。

## 项目简介

`mk-official-notes` 是一个以 HTML 页面为主要内容的存档仓库。其目的在于以结构化方式保存和共享一系列独立的网页文档，适用于需要长期维护、版本可追溯的静态页面集合。仓库本身不对页面内容所属的域名或网站做具体限定，仅作为页面文件的存储和分发载体。

## 目录说明

仓库目录结构如下（示例，以实际为准）：

```
/
├── index.html          # 仓库首页或导航页（可选）
├── pages/              # 主要页面存放目录
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 公共资源（如 CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有独立的 HTML 页面文件，命名建议保持清晰、有规律。
- `assets/`：存放页面中引用的共用样式、脚本或图片资源，便于维护和复用。
- `index.html`（如有）：可作为目录页或入口页面，方便浏览者快速定位。

## 页面归档说明

- 每个 HTML 页面应保持独立、自包含，尽量减少对外部资源的依赖。
- 若页面需要引用公共资源（如样式或脚本），建议统一放置在 `assets/` 目录下，并在页面中通过相对路径引用。
- 页面内容应遵守开源社区规范，不包含恶意代码、侵权内容或违反法律法规的信息。
- 归档的页面版本可以通过 Git 提交历史追溯，便于回滚或查看变更。

## 维护说明

- 本仓库由维护者定期更新，新增页面或修正已有页面内容。
- 欢迎通过 Issue 提交建议或报告问题，但请勿提交与仓库存档无关的内容。
- 若需贡献新页面，请确保页面文件格式正确、资源引用路径无误，并通过 Pull Request 提交。
- 仓库不提供对外服务的 SLA 承诺，页面可能因维护或调整而临时不可访问。
